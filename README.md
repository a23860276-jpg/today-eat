# 🍽️ 今天吃什麼？

幫你和家人快速決定三餐要吃什麼的小工具。先把常去的店家建好，設定當天條件，按一個大按鈕，App 就**隨機跳出一間**當答案，並且預設**避開最近吃過的**。

- 純前端、單一檔案（`index.html`），**不需要安裝、不需要後端、不需要登入**。
- 資料**存在你自己的瀏覽器**（localStorage），可匯出／匯入 JSON 備份。
- 手機優先、繁體中文介面。

---

## 怎麼在電腦上打開

最簡單：**用 Chrome 開啟 `index.html`** 即可（雙擊檔案，或把檔案拖進 Chrome 視窗）。

> 小提醒：用 `file://` 直接開啟時，少數瀏覽器（或無痕模式）會限制本機儲存，導致資料無法保存。
> 如果遇到「儲存失敗」或資料不見，改用下面的小型本機伺服器最穩定：

```bash
cd "今天吃什麼"
python3 -m http.server 8000
```

然後打開瀏覽器到 <http://localhost:8000> 。（macOS 內建 `python3`，不需另外安裝。）

---

## 怎麼在手機上用

這是一個網頁 App，手機上有兩種用法：

1. **最簡單**：把 `index.html` 傳到手機（AirDrop／雲端硬碟／Email），用手機瀏覽器開啟。
2. **像 App 一樣**（建議）：把這個檔案放到任何免費靜態網站空間（例如 GitHub Pages、Netlify），
   用手機瀏覽器打開後選「**加入主畫面**」，就會變成一個全螢幕的圖示 App。
   - iPhone：Safari →分享鈕→「加入主畫面」
   - Android：Chrome →右上選單→「加到主畫面」

> 注意：手機和電腦的資料**各自獨立**（都存在各自的瀏覽器）。要把店家搬到另一台裝置，用下方的「匯出／匯入」。

## 使用流程

### 1. 建立店家（「店家」分頁）
- 右上「＋ 新增」，填**店名（必填）**，其餘屬性用**點選標籤**設定：
  - **主食**（可多選）：麵、飯、鍋物、麵食、其他
  - **菜系**（可多選）：中式、西式、日式、東南亞、韓式、其他
  - **餐別**（可多選）：早餐、午餐、晚餐、宵夜
  - **距離**：步行可到 ～ 車程30分鐘以上
  - **價位**：100以內、100~200、200~300、300~500、500以上
  - 再加上地址、備註。
- **兩人投票**：每張店家卡片下方有兩排按鈕（名字可在設定改），兩個人可各自標記**喜歡／普通／不喜歡**；再點一下同一個可取消。
- **第一次打開 App 時，會自動帶入嶺東／南屯一帶的人氣店家**當作起始資料，可自由編輯或刪除。想重新帶入可到「設定 →重新帶入在地起始店家」。
  - 起始資料的車程是**概估值**，請依你的實際情況調整（尤其「距離」是相對你家的車程）。

### 2. 設定今天的條件（「決定」分頁）
- 大部分標籤是**點一下＝要**（綠）、**再點一下＝不要**（紅，排除）、**再點＝不限**。沒選的就不限制。例如把「麵」設成不要、把「晚餐」設成要。
- **距離**比較特別：它是「**最遠可接受**」的範圍——選一個就好，會**自動包含更近的**。例如不想開太遠就選「**車程10分鐘內**」，步行可到、車程5分鐘內的店也都會被算進來。
- 開啟時會依目前時間自動幫你選好餐別。
- 可開關「**避開最近吃過的**」與「**排除有人不喜歡的店**」（用上面店家卡片的投票結果，只要任一人按過「不喜歡」就不抽）。

### 3. 決定！
- 按底部大大的「**決定！**」，會跳出**一間**店（可選老虎機動畫）。
- **兩人否決權**：結果上有「**○○ 不喜歡**」兩個鈕（名字可在設定改）。**每人每一輪各能按一次**，按下去就自動換一間（會避開剛被否決的店）。兩人都用掉後就只剩「就吃這間」，避免一直重抽。想先看位置可按「🗺️ 看地圖」。
- 決定了按「**就吃這間！**」——會記入紀錄，並**自動開啟 Google 地圖導航**到那間店（可在「設定」關掉）。
- 沒有符合的店時，會提示你放寬條件。

## 備份與還原（換手機／怕資料不見）

到「**設定 →資料備份**」：

- **匯出 JSON**：下載一份備份檔（檔名含日期）。
- **匯入 JSON**：選備份檔還原，**會覆蓋**目前資料（匯入前會再確認一次）。
- **合併匯入店家**：選一個清單檔，把裡面的店家**加進來、不覆蓋**現有資料；店名重複會自動略過。可重複按、不會產生重複。
- **清除所有資料**：把店家、設定、紀錄全部清空（建議先匯出備份）。

### 用 JSON 匯入餐廳清單（給之後持續新增用）

把餐廳整理成下面格式的 `.json` 檔，到「**設定 →合併匯入店家**」選檔即可。**合併匯入只會「新增」，不會覆蓋你改過的店家**（店名相同就自動略過），可以重複匯入同一個檔。

附帶的 [`餐廳清單.json`](餐廳清單.json) 就是 45 間的範本（也是 App 預設帶入的清單），直接複製一筆改一改最快。

**格式：**
```json
{
  "version": 1,
  "stores": [
    { "name": "店名（必填）",
      "staple": ["飯", "麵"],
      "cuisine": ["日式", "中式"],
      "meals": ["午餐", "晚餐"],
      "distance": "車程10分鐘內",
      "price": "100~200",
      "note": "備註（可留空）" }
  ]
}
```

寫起來很彈性，以下都吃得下：
- **多選欄位**（主食／菜系／餐別）可以用陣列 `["飯","麵"]`，也可以直接用頓號字串 `"飯、麵"`（逗號、斜線也行）——**直接從 Excel 儲存格貼過來就能用**。
- **欄位名稱**可用英文（`name/staple/cuisine/meals/distance/price/note`）或中文（`餐廳/主食/菜系/餐別/距離/價位/備註`）。
- 各欄位的可選值請對齊上面「建立店家」列出的標籤（例如距離要寫「車程10分鐘內」這種）。沒填的欄位留空即可，之後在 App 內補。

## 其他功能
- **一鍵自動算距離**：「設定 →距離設定」設好「家的位置」、按「開始」，就會用 Google 幫**還沒填距離**的店家推算距離。**只會存「車程X分鐘內」級距、不保留座標**；查不到的店會列出並略過；**不會覆蓋**已填好的距離。
  - Google 金鑰**藏在後端代理（Cloudflare Worker）**，不會出現在前端網頁——所以這個 App 可以公開給別人用、使用者不用自備金鑰。部署方式見下方。
  - 目前是**直線距離估算**；之後會升級成 OSRM／OpenRouteService 的**實際車程**（程式已預留切換點）。
- **一鍵自動分類（AI 判讀）**：「設定 →自動填寫」按「開始」，AI（OpenAI，會**上網查證**）幫**還沒分類**的店家填「主食／菜系／餐別／價位」（不含距離）。只填空欄、**不覆蓋**你已填的；AI 不確定的會略過。**AI 是參考、可能有誤，填完請逐一檢查**（尤其價位）。一樣走後端代理、金鑰藏在 Worker。
- **兩人否決權**：可在「設定 →否決權成員」改兩個人的名字，讓這個 App 也能給別人用。
- **Google 地圖導航**：選定店家後自動開啟 Google 地圖、直接帶到導航；可在「設定 →選定後開啟 Google 地圖」關閉。
- **統計**：設定頁可看「最常被抽到的店」和「最近吃過」清單（可單筆刪除）。
- **避開最近幾次**：可在設定調整（預設 5 次）。

## 技術說明（給之後想維護的人）
- 單檔 `index.html`，內含 HTML / CSS / vanilla JavaScript，無任何相依套件或 CDN。
- 所有資料存在 `localStorage` 的 `eat-what-data` 這個 key，結構為單一 JSON 物件：
  `{ version, stores[], tagOptions, log[], settings }`。店家**不存座標**（算距離時即時取得、用完即丟，只留級距）；`settings.home` 是距離基準點。
- **距離查詢**：前端 `googleGeocode()` 打 `DISTANCE_PROXY`（`index.html` 最上面的常數，填你的 Worker 網址）；Worker（見 `worker.js`）持金鑰、再打 Google Places API (New) 的 `searchText`，回傳 `{result:{lat,lng,name}}`。金鑰只在 Worker、不在前端。
- **距離換算**集中在 `estimateDriveMinutes(home, s)`：目前回傳「直線公里 × `MIN_PER_KM`」，要換**實際車程**只改這個函式（改成 `await` 呼叫 OSRM `router.project-osrm.org/route/v1/driving/...` 取 `duration/60`，或 OpenRouteService），其他都不用動——檔案裡有註解範例。
- **AI 分類**：前端 `aiClassify()` 打同一個 Worker（body 帶 `op:"classify"`）；Worker 用 OpenAI Responses API（`web_search` 工具）查證後回傳**限定在合法標籤內**的 `{staple,cuisine,meals,price}`。只存推論出的標籤、不存查到的原始內容。
- **廣告／付費解鎖掛鉤**：兩顆自動按鈕都會先 `await requireUnlock(feature)`（目前一律放行）。要放廣告或付費牆，就改這一個函式即可（回傳 `false` 即擋下該功能）。
- 之後想加新功能，資料結構已預留空間，直接擴充即可。

## 設定後端代理（Cloudflare Worker）
金鑰（Google + OpenAI）都藏在 Worker，使用者不必自備。一次設定，之後都不用再碰。

**前提**：
- Google API 金鑰（Cloud Console → 啟用 **Places API (New)** → 開通計費 → 建立金鑰）— 給「自動算距離」用。
- OpenAI API 金鑰 — 給「自動分類」用。

**部署 Worker：**
1. 註冊 [Cloudflare](https://dash.cloudflare.com/)（免費）。
2. **Workers & Pages → Create → Worker** → 命名（例 `eat-dist`）→ Deploy。
3. **Edit code**，把專案裡 [`worker.js`](worker.js) 的內容整個貼上 → Deploy。
4. **Settings → Variables and Secrets** 加：
   - `GOOGLE_KEY`（**Secret**）= 你的 Google API 金鑰
   - `OPENAI_KEY`（**Secret**）= 你的 OpenAI API 金鑰
   - `ALLOWED_ORIGIN`（**Text**）= 你的網站，例：`https://a23860276-jpg.github.io`（只到網域、不要斜線）
5. 複製 Worker 網址（像 `https://eat-dist.xxxx.workers.dev`）。
6. 把網址填到 `index.html` 最上面的 **`DISTANCE_PROXY`** 常數，存檔、Push。

**安全（必做）**：
- Google Cloud → Places API (New) → **配額** 設每日上限（被亂用時的最後防線）。金鑰原本的「網站」限制可保留不動（Worker 會帶對應的 Referer）。
- OpenAI → Usage limits 設每月上限（AI 分類含 web search，比距離查詢貴；若要對外開放，建議在 `requireUnlock()` 加廣告／付費牆來分攤成本）。

> 想換金鑰或改用別家 AI，只要動 `worker.js`、前端完全不用改。
